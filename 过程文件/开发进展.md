# 币安合约横盘监控 — 开发进展

## 核心目标
监控全网 Binance USDT 永续合约中长期横盘、具备爆发潜力的标的，按横盘持续时间降序排列。

## 当前状态：✅ 核心引擎已在云端验证通过

---

## 进展日志

### 第一阶段：调研与方案选型 ✅
*   Coinglass `/zh/volatility` 页面：无法按交易所/合约类型过滤，无"横盘持续时间"指标。
*   Coinglass `/zh/pro/i/MACD` 页面：同样缺乏所需维度。
*   **结论**：必须自建引擎，直接从币安 API 拉取原始 K 线数据做本地计算。

### 第二阶段：网络攻坚 (Proxy Bypass) ✅
*   编写了 `test_binance_proxies.py` 和 `test_binance_proxies2.py` 测试多个公共反代。
*   **结论**：90%+ 公共节点不可用（403/451/超时/仅现货）。
*   用户提供了高质量 Cloudflare Worker 反代代码 (`cloudflare_worker_proxy_advanced.js`)，绑定自定义域名 `binance.794988.xyz`。
*   **但实测 Worker 依然返回 451**：因为 Cloudflare 的 `fetch` 出口 IP 仍被币安识别为云服务商封禁。
*   **最终方案：Tor 匿名网络（洋葱路由）**
    *   在 GitHub Actions 云端安装 Tor 服务。
    *   强制出口节点锁定为：德国/法国/日本/新加坡/英国/荷兰/瑞士/瑞典/韩国（9 国）。
    *   Python `requests` 通过 `socks5h://127.0.0.1:9050` 隧道发包。
    *   **验证成功**：出口 IP `45.138.16.249`（德国），`{"IsTor":true}`，币安返回 200。

### 第三阶段：引擎构建 (Sideways Scanner) ✅
*   核心文件：`sideways_scanner.py`
*   **横盘判定算法（向后追溯法）**：
    *   从最新 K 线开始往历史追溯。
    *   不断扩大 `[最高价, 最低价]` 区间。
    *   一旦区间振幅 `(Max - Min) / Min > 2%`，停止计时。
    *   输出：该标的当前连续横盘的 K 线根数（即小时数）。
*   **并发能力**：`ThreadPoolExecutor(max_workers=20)`，544 个合约全部扫完约 10 秒。
*   **容错机制**：`exchangeInfo` 请求内置 3 次重试（间隔 15 秒），超时 60 秒。

### 第四阶段：云端部署 ✅
*   GitHub 仓库：`https://github.com/w20gb/binance-proxy`
*   分支：`main`
*   Workflow 文件：`.github/workflows/sideways_monitor.yml`
*   **Tor 启动保障**：主动轮询 `check.torproject.org/api/ip` 确认 `IsTor:true` 后才启动引擎（最多等 120 秒）。
*   **报告输出**：
    *   直接打印到 Actions 日志（展开即看）。
    *   同时上传为 Artifact（保留 30 天，可下载）。
*   **定时执行**：每天北京时间 12:00 和 20:00 自动巡检。

## 关键配置参数 (sideways_scanner.py)
| 参数 | 当前值 | 含义 |
|---|---|---|
| `INTERVAL` | `1h` | K 线级别 |
| `LIMIT` | `200` | 最大追溯 K 线数（约 8 天） |
| `AMPLITUDE_THRESHOLD` | `0.02` (2%) | 横盘振幅收敛上限 |
| 最低横盘门槛 | `> 6 小时` | 报告中过滤掉短于 6 小时的标的 |
